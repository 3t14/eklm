# Core Novelty Claim

更新日: 2026-03-28

## 一文で言うと

本構想の中核的新規性は、

`知識を大量に内在化した言語モデル` ではなく、
`脱語彙化された構造を学習し、知識は後から接続する language model`

を、モデル原理として正面から置く点にある。

## 何が新しいのか

単に

- interlingua を使う
- semantic parsing を使う
- constrained decoding を使う
- 外部知識を使う

というだけなら、既存研究の組み合わせに見える。

しかし本構想の主張はそこではない。

主張の芯は、

1. 学習すべき対象を `知識` ではなく `構造` に限定する
2. 内容語や専門語はまず匿名化、抽象化して扱う
3. 入力の各部分を、その意味型に応じて最も安い計算系へ振り分ける
4. 知識は重みではなく外部知識部に置き、必要時だけ接続する

という設計原理そのものにある。

つまりこれは、
`知識を圧縮した巨大モデル`
を作る路線ではなく、
`知識を持たない代わりに構造を強く持つ小型モデル`
を作る路線である。

## 既存研究との差

### 1. semantic parsing との違い

semantic parsing は普通、

- 自然言語を logical form へ写す
- task ごとに target formalism を持つ
- parser 精度を上げる

ことを目的にする。

本構想はそれより広く、

- その写像を言語モデルの中心任務にする
- target を task-specific な logical form ではなく共通IRにする
- さらに知識部、導出部、計算系の振り分けまで含めて設計する

点が違う。

### 2. RAG / 外部知識系との違い

RAG や KB 系は、
`知識を外に出す`
点では近い。

ただし多くは、

- 表現部は依然として通常の Transformer
- 知識取得は text / vector 類似度が中心
- 入力理解そのものの構造化は主題ではない

本構想は逆に、
`知識の外部化` より先に
`入力理解を構造化し、構造ごとに知識接続や導出を変える`
ことを中心に置く。

### 3. grammar engineering / interlingua との違い

GF や UMR に近いのは、

- 共通表現を置く
- 言語差分をモジュール化する

点である。

しかし本構想ではさらに、

- 数式と正規表現を第一級表現として扱う
- Markdown、自然言語、コード、数式を混在文書として扱う
- 曖昧性を残したまま小型モデルで構造割当する
- 知識部と導出部を分離する

ところまで含める。

## いちばん斬新なアイディア

いちばん斬新な点を一つだけ選ぶなら、

`脱語彙化された構造学習を、言語モデルの一次目的にすること`

である。

ここでの脱語彙化とは、

- 助詞
- 記号
- 演算子
- 構文役割
- 型
- 関係
- 量

のような高頻度な構造要素だけを固定的に学習し、

- 固有名詞
- 専門語
- 分野固有記号
- 文脈依存語彙

は最初は `PERSON_SLOT`, `TERM_SLOT`, `VAR_SLOT`, `FUNC_SLOT` のような抽象スロットへ落とすことを指す。

これにより、モデルは
`何について話しているか`
よりも
`それが文や式の中でどう振る舞うか`
を先に学ぶ。

この性質は、

- 小データでも学習しやすい
- 未知語に強い
- ドメイン移植しやすい
- 知識更新を外部化しやすい

という利点に直結する。

## 第二の斬新さ

二つ目の新規性は、

`意味の型に応じて、最も安い計算へ落とす`

という点である。

Transformer は多くの場合、ほぼすべてを同じ token prediction で処理する。

本構想ではそうではなく、

- 表層パターンなら regex / FSA
- 線形関係なら行列演算
- 既知事実なら lookup
- 制約なら constraint solving
- 曖昧な自然言語だけ small Transformer

というように計算系を分業させる。

これは単なる tool use ではなく、
`意味表現の型と計算複雑性を対応づけるアーキテクチャ`
である。

## 研究提案として強い言い方

研究提案としては、次の言い方が強い。

本研究は、知識を重みに圧縮する従来型 language model とは対照的に、脱語彙化された構造のみを学習する表現モデルを中核に置き、知識は外部知識部に分離し、推論は意味型に応じて最小計算系へ振り分ける低コスト language model を提案する。

## 何を証明できれば強いか

この主張を強くするには、次のどれかを実証する必要がある。

1. 同じタスクで学習コストを大幅に下げられる
2. 未知語や新規ドメインに強い
3. 知識更新を再学習なしで高精度に反映できる
4. 小型モデルでも一定以上の精度を出せる
5. 失敗時に `未解決` として止まり、幻覚を減らせる

特に 1 と 3 が出ると、主張がかなり強くなる。

## 実装上の最小仮説

最小プロトタイプとしては、次の仮説が現実的である。

- 自然言語を脱語彙化した共通IRへ写す小型Transformerを作る
- 数式と pattern は deterministic parser で直接 IR 化する
- 知識部は slot 解決と rule lookup だけに限定する
- 導出部は線形代数、単純制約、regex matching に限定する

この最小形でも、

- structure-first
- knowledge-later
- cheap computation routing

の3点は検証できる。

## 結論

この構想の本当の新規性は、単なるハイブリッド設計ではない。

本質は、
`言語モデルはまず知識を学ぶべきだ`
という主流前提を外し、
`言語モデルはまず構造を学び、知識は後から接続すべきだ`
という原理を中心に据えることにある。
